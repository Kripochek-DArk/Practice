# Лабораторная №3

## Задача 1

### Текст задачи

На основе последовательности, содержащей римские обозначения цифр от I до IX, получить их десятичные представления.  

### Алгоритм решения

- создается функция inputNumbers, которая формирует последовательность римских чисел
- последовательность создается с помощью Seq.unfold
- программа запрашивает ввод римского числа от I до IX
- если введена пустая строка — последовательность завершается
- если введено корректное римское число — оно добавляется в последовательность
- если введены некорректные данные — выводится сообщение об ошибке и ввод повторяется
- полученная последовательность римских чисел передается в Seq.map
- Seq.map выполняет преобразование римских чисел в десятичные с помощью функции romanToNumber
- полученные десятичные числа выводятся на экран с помощью Seq.iter

## Задача 2

### Текст задачи

Список содержит десятичные цифры. Составить число из чётных цифр.

Ввод цифр осуществляется с клавиатуры.  
Признаком окончания ввода является пустая строка.

### Алгоритм решения

- создается функция inputNumbers, формирующая последовательность введённых пользователем цифр
- последовательность создается с помощью Seq.unfold
- программа запрашивает ввод цифры от 0 до 9
- если введена пустая строка, последовательность завершается
- если введено корректное значение (число от 0 до 9), оно добавляется в последовательность
- если введены некорректные данные, выводится сообщение об ошибке и ввод повторяется
- полученная последовательность цифр передается в функцию Seq.fold
- Seq.fold последовательно перебирает элементы последовательности
- если текущая цифра является чётной, она добавляется к формируемому числу
- добавление происходит по формуле: текущее число умножается на 10 и прибавляется текущая цифра
- если цифра нечётная, она пропускается
- после обработки всей последовательности выводится полученное число

##Задача 3
Текст задачи

Указать, файлы с каким расширением встречаются в указанном каталоге и его подкаталогах реже всего.

Алгоритм решения

- создается функция getExtensions, которая получает все файлы из указанного каталога и его подкаталогов
для получения файлов используется Directory.EnumerateFiles с параметром SearchOption.AllDirectories
-для каждого найденного файла определяется его расширение с помощью Path.GetExtension
расширение приводится к нижнему регистру с помощью ToLower, чтобы одинаковые расширения в разном регистре считались одинаковыми
-из последовательности исключаются пустые расширения, то есть файлы без расширения не учитываются
-с помощью Seq.countBy подсчитывается количество файлов для каждого расширения
-полученная последовательность пар «расширение – количество» сортируется по количеству с помощью Seq.sortBy
-создается функция findRarestExtensions, которая находит расширения, встречающиеся минимальное количество раз
-если последовательность расширений пуста, выводится сообщение о том, что в каталоге нет файлов с расширениями
-если расширения найдены, определяется минимальное количество вхождений с помощью Seq.minBy
-после этого с помощью Seq.filter выбираются все расширения, количество которых равно минимальному
-на экран выводится путь к каталогу, минимальное количество вхождений и список самых редких расширений
-в главной функции main пользователь вводит путь к каталогу
-если каталог существует, вызывается функция findRarestExtensions
-если каталог не существует, выводится сообщение об ошибке
