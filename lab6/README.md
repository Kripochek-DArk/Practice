# Лабораторная №6

## Задача 1

### Текст задачи

ООП. Разработать класс с двумя логическими полями. Создать конструктор копирования. Разработать метод, вычисляющий эквиваленцию полей. Перегрузить метод ToString() для формирования строки из полей класса. Реализовать дочерний класс (его содержание предложить самостоятельно и описать в решении: какой содержательный смысл имеют поля; реализовать конструкторы; предложить и реализовать 2-3 метода). Протестировать все конструкторы и другие методы базового и дочернего классов.

### Алгоритм решения

- Разработан базовый класс Device с двумя логическими полями, описывающими состояние устройства
- Реализованы конструктор по умолчанию, конструктор с параметрами и конструктор копирования
- Разработан метод, вычисляющий эквиваленцию логических полей через их сравнение
- Переопределён метод ToString() для формирования строки с текущими значениями полей
- Создан дочерний класс SmartLamp, наследующий Device
- В дочернем классе добавлены дополнительные поля, характеризующие объект (яркость и название)
- Реализованы конструкторы по умолчанию, с параметрами и копирования с использованием base
- Добавлены методы для управления состоянием объекта (включение, изменение яркости, получение информации)
- В программе выполнено тестирование всех конструкторов и методов базового и дочернего классов через создание объектов и вывод результатов их работы

## Задача 2

### Текст задачи

Задания 2 и 3 решаются в одном проекте (задание 3 продолжает задание 2).
В задании 2 необходимо реализовать определение класса (поля, свойства, конструкторы, перегрузка
метода ToString() для вывода полей, заданный метод согласно варианту). Протестировать все
методы, включая конструкторы (при тестировании методов классов в заданиях не забывайте
проверять вводимые пользователем данные на корректность).

1)Перегрузка операций
Time byte hours
byte minutes
Вычитание времени (величины типа
Time)из объекта типа Time (учесть, что
возможен переход в предыдущие
сутки).Результат должен быть типа Time.

### Алгоритм решения

- Реализован класс Time с двумя полями: hours и minutes

- Добавлены свойства Hours и Minutes для доступа к полям с проверкой корректности значений

- Реализован конструктор по умолчанию, конструктор с параметрами и конструктор копирования

- Переопределён метод ToString() для вывода времени в формате часов и минут

- В программе выполнено тестирование конструкторов, свойств и метода вычитания с вводом данных с клавиатуры

## Задача 3

### Текст задачи

В задании 3 добавить к реализованному в задании 2 классу указанные в варианте перегруженные
операции. Протестировать все методы.

2)Перегрузка операций 
Time Унарные операции:
-- вычитание минуты из объекта типа Time
Операции приведения типа:
byte (явная) – результатом является количество часов (минуты
отбрасываются);
bool (неявная) – результатом является true, если часы и минуты не равны
нулю, и falseв противном случае.
Бинарные операции:
+ Timet, беззнаковое целое число (лево- и правосторонние операции) –
добавление минут к времени.
+ Timet1, Timet2 – сложить два времени.

### Алгоритм решения

- В класс Time добавлена перегрузка унарного оператора -- для уменьшения времени на одну минуту

- Реализовано явное приведение типа к byte, которое возвращает только количество часов

- Реализовано неявное приведение типа к bool, которое возвращает false для времени 00:00 и true для любого другого значения времени

- Добавлена перегрузка бинарного оператора Time + uint для прибавления заданного количества минут ко времени

- Добавлена перегрузка бинарного оператора uint + Time для правостороннего сложения минут и времени

- Добавлена перегрузка бинарного оператора Time + Time для сложения двух объектов времени

- Во всех операциях используется перевод времени в общее количество минут с последующим приведением результата к диапазону одних суток

- После вычислений результат переводится обратно в часы и минуты и возвращается как новый объект Time

- В программе выполнено тестирование всех перегруженных операций с вводом исходных данных с клавиатуры
